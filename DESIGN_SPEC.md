# 《废星拾荒者》开发版策划案

## 1. 定位

《废星拾荒者》是一款单机网页放置/增量游戏。玩家扮演一台拾荒机器人，在荒废星球上持续外出探索、自动拾荒、自动战斗、回收垃圾、升级补给站与机器人模块，并通过星球完成度解锁更高级星球。

核心体验不是复杂操作，而是“返航后有大量东西可处理”：

- 新垃圾带来图鉴发现感。
- 材料推动设施升级。
- 装备改变下一次探索效率。
- 遗物和彩蛋提供世界观与惊喜。
- 完成度推动区域和星球解锁。

## 2. MVP 参数

| 项目 | 第一版设定 |
|---|---:|
| 初始外出时长 | 20 分钟 |
| 早期外出时长 | 30-60 分钟 |
| 中期外出时长 | 2-6 小时 |
| 后期外出时长 | 8-24 小时 |
| 事件触发间隔 | 5-20 秒随机 |
| 第一星球区域 | 6 个 |
| 第一版道具 | 300+ |
| 敌人 | 24 个 |
| 设施 | 10 个 |
| 存档 | localStorage |
| 离线收益 | 初始 4 小时，上限随设施提升 |

## 3. 核心循环

```text
补给站整备
-> 选择星球区域
-> 外出拾荒
-> 每 5-20 秒触发随机事件
-> 获得道具 / 遭遇敌人 / 发现遗物 / 危险事件
-> 自动返航
-> 出售垃圾、装备模块、升级设施、查看图鉴
-> 解锁新区域或新星球
```

## 4. 机器人属性

| 属性 | 作用 |
|---|---|
| `hp` | 当前耐久，归零则强制返航 |
| `maxHp` | 最大耐久 |
| `attack` | 自动战斗伤害 |
| `defense` | 减少战斗伤害 |
| `baseExploreMinutes` | 基础外出时间 |
| `cargoSlots` | 单次探索可携带数量 |
| `rareFind` | 稀有物发现率 |
| `hiddenEvent` | 隐藏事件和彩蛋发现率 |
| `gatherYield` | 材料和普通掉落数量加成 |
| `junkValue` | 垃圾出售价值加成 |
| `xpGain` | 战斗经验加成 |
| `escapeChance` | 失败时保留收益概率 |

## 5. 第一星球：锈蚀母星

主题：第一颗废弃殖民星，适合教学和建立长期目标。它由补给站外围、锈蚀荒原、废弃能源设施、坍塌工业城市、地下维修系统和旧发射场组成。

区域设计：

| 区域 | 定位 | 解锁 |
|---|---|---|
| 补给站外围 | 教学区，低风险、低收益 | 默认 |
| 锈蚀平原 | 基础挂机区，垃圾和低级材料丰富 | 默认 |
| 废弃太阳能农场 | 电池、雷达、隔热材料来源 | 星球完成度 18% |
| 坍塌机械城 | 装备和中级材料核心来源 | 击败补给站门禁主机 |
| 地下维修通道 | 芯片、维修、异常物开始出现 | 星球完成度 42% |
| 旧时代发射场 | 第一星球终局区域，解锁下一星球 | 星际导航仪 2 级 |

下一星球解锁：

```text
锈蚀母星完成度 >= 75%
星际导航仪 >= 3 级
拥有“短程星际推进器蓝图”
```

## 6. 事件系统

每次探索按照区域事件权重抽取事件类型，再从对应掉落池中抽取内容。

事件类型：

| 类型 | 说明 |
|---|---|
| `junk` | 获得只能出售的垃圾 |
| `material` | 获得升级材料 |
| `enemy` | 进入自动文字战斗 |
| `equipment` | 获得装备模块 |
| `consumable` | 获得一次性 buff 道具 |
| `relic` | 获得遗物，提升完成度和剧情线索 |
| `blueprint` | 解锁设施或装备制作 |
| `anomaly` | 获得改变规则的异常物 |
| `hazard` | 损失耐久、电量、货物或触发负面日志 |

## 7. 自动战斗

战斗为纯文本自动回合制，玩家不需要操作。

回合流程：

```text
机器人行动
-> 敌人行动
-> 状态结算
-> 判断胜负
```

基础伤害公式：

```text
damage = max(1, attack - defense * 0.5) * random(0.85, 1.15)
```

胜利奖励：

- 经验。
- 敌人掉落材料。
- 低概率掉落装备、蓝图或特殊道具。
- 首次击败提升图鉴完成度。

失败惩罚：

- 强制返航。
- 默认保留 70% 已拾荒物。
- 耐久需要维修。
- 不建议第一版设计成高惩罚，否则会破坏挂机体验。

## 8. 道具分类

完整道具表见 `data/items.json`。

| 分类 | 用途 |
|---|---|
| `junk` | 只能出售，数量最多，承担发现感和图鉴收集 |
| `material` | 设施升级、装备制作、蓝图合成 |
| `equipment` | 装备到机器人槽位，改变探索和战斗效率 |
| `consumable` | 单次探索 buff 或返航补给 |
| `relic` | 世界观、任务、永久加成、图鉴完成度 |
| `blueprint` | 解锁设施、装备或主线制作项 |
| `anomaly` | 改变规则的稀有物，强力但可能有副作用 |

## 9. 装备槽位

| 槽位 | 作用 |
|---|---|
| `weapon` | 提升战斗能力 |
| `shell` | 提升耐久、防御和环境抗性 |
| `battery` | 提升外出时长 |
| `radar` | 提升稀有发现、隐藏事件 |
| `arm` | 提升采集效率 |

| `cargo` | 提升货舱容量 |
| `special` | 提供特殊规则加成 |

## 10. 补给站设施

完整设施表见 `data/facilities.json`。

第一版设施：

- 回收机
- 充电桩
- 仓库
- 工作台
- 雷达塔
- 维修仓
- 研究终端

- 星际导航仪

每个设施 5 级。设施升级消耗废币、材料，部分等级需要蓝图。

## 11. 完成度

完成度用于解锁区域和下一星球。

| 来源 | 完成度 |
|---|---:|
| 首次发现垃圾 | +0.2% |
| 首次发现材料 | +0.5% |
| 首次发现装备 | +1% |
| 首次发现消耗品 | +0.8% |
| 首次发现遗物 | +2% |
| 首次发现蓝图 | +1.5% |
| 首次发现异常物 | +3% |
| 首次击败普通敌人 | +1% |
| 首次击败 Boss | +5% |
| 区域里程碑 | +3% |

实际实现时，完成度建议设置上限为 100，并允许内容总点数超过 100，避免玩家必须全收集才能推进。

## 12. 离线收益

第一版建议按当前“已解锁且推荐战力最高”的区域结算。

```text
有效离线时间 = min(离线时长, 离线收益上限)
离线探索次数 = 有效离线时间 / 平均事件间隔
实际奖励 = 在线掉落期望 * 离线效率
```

初始：

```text
离线收益上限：4 小时
离线效率：40%
```

自动派遣台升级后：

```text
离线收益上限：8 / 12 / 16 / 20 / 24 小时
离线效率：45% / 55% / 65% / 75% / 85%
```

## 13. 前端实现建议

界面建议分成 7 个主标签：

- 补给站
- 探索
- 背包
- 装备
- 设施
- 图鉴
- 日志

第一版代码结构：

```text
index.html
src/state.js
src/data-loader.js
src/explore.js
src/combat.js
src/inventory.js
src/facilities.js
src/collection.js
src/save.js
data/*.json
```

关键原则：

- 所有内容尽量数据驱动。
- 新星球以新增 JSON 为主。
- 探索不要只依赖定时器，也要支持时间戳补算。
- 日志是核心反馈，应当有足够文本变化。
